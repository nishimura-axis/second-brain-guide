# アド管理 業務内容

> 注意: 業務内容は変更される可能性があるため、最新運用と差分がある場合はこのメモを更新する。

## ミッション

広告本部の火力最大化。運用者がクリエイティブ制作、分析、改善に集中できる環境を作る。

## アド管理の役割

- ノンコア業務の巻き取り
- 業務の自動化
- 依頼フローの構築
- 運用者がコア業務に集中できる状態づくり

```text
案件選定 -> 制作 -> つなぎこみ -> 出稿 -> 運用 -> 分析 -> 数字合わせ
                     ↑                         ↑
                   アド管理                   アド管理
```

## 優先順位の考え方

| 分類 | 定義 | 例 |
|------|------|----|
| 赤 | やらないと損失が生まれる、かつ緊急度が高い | 分析用シートのエラー、媒体エラー |
| オレンジ | やらないと損失が生まれるが、緊急度はやや低い | リンク発行、アカウント開設、定常業務 |
| 黄色 | やらなくても回るが、生産性が高まる | 出稿フロー整備、自動化、マニュアル化 |
| 青 | やらなくても困りにくい | 使わなくなったページやシートの整理 |

## 業務カテゴリ

### 1. つなぎこみ

広告配信開始までの技術的なセットアップ全般。

- リンク発行
- 広告アカウント開設
- タグ設置依頼
- 計測連携設定
- 発火確認

### 2. 数字合わせ

月次の売上確定業務。

- 媒体管理画面とASP管理画面の突合
- 差異確認
- 売上確定に必要なデータ整理

### 3. 出稿部管理

運用者からの出稿依頼を受け、出稿作業を円滑に回す。

- 依頼振り分け
- キャパシティ管理
- 品質管理
- 教育
- フロー改善

### 4. スプレッドシート管理

運用に必要なシートの保守運用。

- 月別売上シート
- チーム別運用シート
- 精査用シート
- エラー対応
- 既存シートの改善

### 5. 自動化

業務効率化のための仕組みづくり。

- GAS
- GCP
- RPA
- 業務ツール
- マニュアル作成

## Codexに参照してほしい判断軸

- 運用者の制作・分析時間を増やせるか
- 放置すると損失が出るか
- 属人化を減らせるか
- 定型化・自動化できるか
- 赤、オレンジ、黄色、青のどこに当たるか
