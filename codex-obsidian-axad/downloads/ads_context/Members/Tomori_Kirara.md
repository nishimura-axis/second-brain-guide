## 特性サマリー

### 💪 強み

| 強み | 具体例・根拠 |
|------|-------------|
| **「まずはやってみる」精神** | 新しい領域のタスクも積極的に巻き取って進める。運用を自分でやってみたいと提案 |
| **やると決めたらすぐ手を動かせる（推進力）** | FBバルク生成ツールを1から作成。依頼を受けたらすぐ着手してボールを手放す |
| **GAS/スプシ/AIを活用した業務効率化スキル** | CRチェック依頼自動化（月約80時間削減）、精査シート差異通知、アナウンスログ蓄積など多数実装 |
| **「どうしたらもっとラクになるか」を考えて動ける** | 出稿依頼シートの改修や通知自動化、タスク依頼のフォーム化など「そもそものやり方を変える」動き |
| **新しい領域にも臆せず挑戦する姿勢** | チャットボット作成への挑戦、AIツール選定、新規ASP対応など |

### 🔧 弱み・課題

| 弱み | 具体的な事象 |
|------|-------------|
| **言語化能力** | やっていること・考えていることを人に伝えるのが苦手。WBS作成、プロジェクト進捗の説明に時間がかかる |
| **スケジュール管理** | 見積もりが甘い、リスケのタイミングが遅れがち。スケジュール通りに進めることができなかった |
| **上流工程（全体像把握→課題発見→要件定義）** | 来た依頼をこなすことはできるが、全体像を見て現場からの吸い上げが足りない |
| **共有・ログ残しの不足** | 口頭コミュニケーションで済ませがち。自分の中で完結して共有が不足 |
| **人の力を借りる、人を動かすこと** | どうしても自分で手を動かしたくなる。相談するタイミングが遅れる |

### 🧠 個人特性

| 特性 | 説明 |
|------|------|
| **単純作業が苦手** | 定期タスクに時間がかかりすぎる傾向。効率化できないかの意識で向き合う |
| **手を動かすのが好き・得意** | 頭の中で考えてすぐ手を動かすことが得意。これが自分の長所と自認 |
| **言語化・WBS作成が苦手** | プロジェクトの全体像と進捗をわかりやすく伝えることに苦手意識 |
| **新しいことへのアンテナ** | AI、ツールなど効率的なものに目を向ける意識あり |
| **コミュニケーションを大事にする** | 出稿部との関係構築、運用者との直接コミュニケーションを増やす姿勢 |

## FY2025（2025年4月〜2026年3月）

### Q1（4-6月）

#### 📎 成果目標

| No | 目標 | 達成基準 | ウェイト |
|----|------|----------|----------|
| 1 | 既存案件・既存媒体のつなぎこみの自走 | 全媒体引継ぎ進捗シートで各項目の80%以上は自走完了 | - |
| 2 | 運用者が最小限の工数で出稿できる仕組みを作る | 出稿リソース不足で翌日に持ち越すキャンペーンを0にする | - |
| 3 | 既存スプシ周りの業務引継ぎ、GAS等新技術領域に挑戦 | 既存スプシの依頼の宛先を友利にする、業務効率化するタスクを5個実施 | - |

#### 📝 期末振り返り

**達成度**: 達成

**総括**:
- 前Qは会社に慣れる、仕事に慣れるがメインだったが、今Qは色々学べてできることも増えた
- スプシやGAS周りの知識、運用の知識が一番インプットして伸びた
- この会社に入ってよかったと実感

**得られた成果**:

**◇つなぎこみ**
- 既存つなぎこみの宛先を自分にしたことで、責任もって対応することができた
- 自分ボールを無くす、期限を考えて動く意識がついた
- 過去のやり取りを見つつ、すぐ聞いて解決。次のアクションをもって聞くことを心掛けた
- つなぎこみでできることが増え、先を見据えて動くことが少しずつできるようになった
- メンション付いていないASPとのやり取りで、依頼が来る前に相談することができた

**◇スプシ、GAS**
- 既存スプシの宛先を自分にすることでスプシやGASに触れる機会が増えた
- chatgptや既存の仕組みを参考に、自分で考えて対応することができた
- 業務効率化タスクを5個実施
  - 出稿依頼シート改修
  - 出稿依頼シート自動通知
  - タスク依頼フォーム化
  - CSV出力稼働通知
  - キャップ通知設定

**◇運用**
- タスクが落ち着いたタイミングで、運用を自分でできる機会ができた
- 運用について1日3人以上に自分から質問しに行くを心掛けた
- 運用者への出稿レクチャーを出稿依頼の部分からレクチャーに変えることができた

**残った課題**:
- スピード感が足りない。夜になって今日何やったけと思う日が増えた
- 定期タスクに時間がかかりすぎた
- 出稿のリソース不足を大きく改善することができなかった
- インプットは増えたがアウトプットができていない

**次Qへの申し送り**:
- スピード感とアウトプットの質を向上
- 「広告本部の攻撃力最大化」「属人化からの脱却」の視点を持つ

**達成度**: 目標1達成、目標2一部達成

**総括**:
- 今Qは自分の長所に気づいて伸ばす、伸ばしてもらうことができた

**得られた成果**:

**◇新規案件つなぎこみ**
- 前Qでは既存のつなぎこみを点から線にしたが、今回は新規案件が始まるところから業務を覚えることができた

**学び**:

- ツールの選定、AIの得意分野/使い分け

**残った課題**:
- チャットボットAI試作を作ることができなかった
- スケジュール通りに進めることができなかった（見積もりが甘かった）
- 人の力を借りる、人を動かすの部分が足りなかった
- 言語化して伝えることが苦手と明確になった

**次Qへの申し送り**:

- 人に説明して理解してもらえるレベルまでモノゴトを構造的に捉える
- アド管理のエースプレイヤーとしての業務推進（既存領域をイレギュラー含め完全自走）
- 既存領域の業務のマニュアル化

#### 📋 行動目標

##### 目標1：業務効率化（メイン）

〈KGI〉
現場の業務効率化により、運用者・出稿部の稼働を最大化した状態

〈行動目標〉
・スプシ/ツールを使用した業務効率化・改善の仕組みを開発
・運用者の出稿に割く時間を最小限にする施策を実行
・出稿部の稼働を最大化する仕組みを構築

〈達成基準〉
・業務改善のスプシ/ツールを5つ作成
・運用者の出稿リソースを1人当たり1日30分削減
・出稿部の稼働効率が向上した定量データを提示

---

##### 目標2：能力開発

〈KGI〉
アド管理の業務に再現性を持たせ、属人化を脱却した状態

〈行動目標〉
・全員ができるように業務のマニュアル化を推進
・誰がやっても同じ成果を出せるやり方を設計・作成
・自身の持つスキル（GAS/スプシ/AI活用）を型化して共有

〈達成基準〉
・マニュアル化された業務が3つ以上
・マニュアルを使って他メンバーが自走できた事例を提示
・属人化していた業務が他メンバーに移管完了

---

##### 目標3：課題発見/対応

〈KGI〉
現場の声を継続的に吸い上げ、ツールを改善し続ける仕組みがある状態

〈行動目標〉
・週1回、新入社員/管理職にヒアリングを実施
・導入後のツールについて使用者にヒアリングし、ブラッシュアップ
・改善要望をリスト化し、優先度をつけて対応

〈達成基準〉
・週1回のヒアリングが12週継続（1Q通して）
・ツールのブラッシュアップ履歴を記録し、改善回数を可視化
・ヒアリングから生まれた改善施策を3件以上実行
