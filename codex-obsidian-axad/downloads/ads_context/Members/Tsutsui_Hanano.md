### 🔧 弱み・課題

| 弱み | 具体的な事象 |
|------|-------------|
| **タスク管理・時間管理** | 目の前のタスクに追われ、長い目でスケジュールを見れない。毎日タスク整理に時間をかけている |
| **完璧主義・規範意識の強さ** | 柔軟性に欠け、スピードより「きれいさ」を優先しがち。取捨選択が苦手 |
| **感情で動く傾向** | 事実と解釈のすみわけができず、感情で仕事をしてしまう場面がある |
| **言語化力** | 考えていることを言葉にするのが苦手。提案時もわかりづらいと指摘されることがある |
| **視座を上げること** | 「マイナス→ゼロ」の動きは得意だが、「ゼロ→プラス」の動きに踏み込む余裕がない |
| **余白を作ること** | 自分のリソースを空ける動きが苦手。抱え込みがち |

### 🧠 個人特性

| 特性 | 説明 |
|------|------|

| **規範意識が強い** | ルールや決まりごとを大切にする反面、柔軟な対応が難しい |
| **内省的** | 自分の内面と向き合うことを意識的に行っている。日々ざらつきがあれば感情や事実を探る動きが習慣化 |

| **やる気を待つメンタルモデル** | モチベーションに左右されやすい自覚あり |

### Q1（4-6月）

#### 📎 成果目標

| No | 目標 | 達成基準 | ウェイト |
|----|------|----------|----------|
| 1 | 状態目標①：案件状況、媒体の最新動向を把握し、滞りなく運用できる状態を作る | つなぎこみ要因での運用開始後ろ倒しゼロ | - |
| 2 | 状態目標②：現場の課題を解像度高く把握し、広告本部の攻撃力最大化につながるタスクを自ら発生させる | 自ら生み出したタスク4個以上 | - |
| 3 | 状態目標③：自分の担当領域の属人化を脱却するために仕組みを構築できる | 自ら作り出した仕組み3個以上（運用できている状態） | - |

#### 📝 期末振り返り

**達成度**: 70-80%

**総括**:
- 部としては広告本部の拡大に合わせて、状態がよくない中耐えしのいだ
- 個人としては、現状のまま現場を回し続ける・推進する力が向上
- 自分の内面と向き合うのを頑張った3か月

**得られた成果**:
- ①達成：つなぎこみ要因での後ろ倒しはなし
- ②達成率70-80%：常にタスクを発生させることはできた
  - Xアカウント状況シート作成
  - エンターキー消化金額のCSV集約（システム構築）
  - 数字合わせシートブラッシュアップ
  - Notionタスク管理検証
  - 予備端末管理の作成＠管理部連携
  - メンクリ/ストのコード共有シート記載方法すり合わせ
- ③達成率70-80%：仕組み作りにも携われた
  - 新入社員入社時のフロー構築（BO連携）
  - CSV保管修正（インセン計算のため）
  - BO連携MTGの進め方
  - ログ残しチャット作成

**残った課題**:
- 「広告本部の攻撃力最大化」を念頭に置いてタスク発生できていなかった
- 目の前の課題を拾っただけで、視座が上がっていない
- 日々のタスクのコントロール、時間のコントロールが不十分
- セルフマネジメント（やる気を待つ、一つのことを引きずる、解釈で動く）

**次Qへの申し送り**:
- タスク管理を一元管理する
- タスクのすみわけ、取捨選択を意識
- 「時間を作る」「余裕を作る」意識で働く
- インプット時間を作る（イシューからはじめよう、エッセンシャル思考）

#### 💬 西村コメント

> その上で、やっぱり目の前の整理に時間を取られすぎて、もう一歩先に踏み込む余裕を作れなかったのが一番の課題だったという点は同意です！

**得られた成果**:
- **チームとして**
  - 精査・出稿自動化＠西さん
  - 運用者の業務効率化（CRのチェック依頼自動化、CR制作依頼シート等）

- **個人として**
  - BO連携強化（入社・退職フロー構築、TikTok請求承認フロー化、入社前フォローアップのブラッシュアップ）

**学び**:
- スピード感の重要性（意思決定の速さ、方向転換に振り落とされないようにしないと推進していたタスクが無駄になる）

**残った課題**:

- つなぎこみ、出稿領域のインプットの抜け

**次Qへの申し送り**:
- 広告本部の動向の察知（スピード感や判断軸に影響）
- 柔軟性の向上（スピードを念頭に置いて、基準や「きれいさ」を時に捨てる判断）
- 改めて触れてない業務や、長期間触ってない業務で理解を曖昧なままにしてる部分を振り返る
