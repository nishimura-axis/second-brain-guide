# 講義コンテキスト作成ガイドライン

## 概要
新入社員オンボーディング用のアドアフィ基礎講義動画（全10本）の文字起こしを整理・要約し、コンテキストとして保存するためのルール。

## ファイル形式
- **保存先**: `20_Context/AXIS/Ads/`
- **命名規則**: `Lecture_XX_[タイトル].md`
- **タイトル**: 内容から提案 → ユーザー承認後に確定

## 各ファイルに含める情報
| 項目 | 内容 |
|------|------|
| タイトル | 内容から提案 → 承認後確定 |
| 撮影日 | 必須記録（不明な場合は質問） |
| 要約 | 文字起こしの内容を正確に反映 |

## 要約時の厳守事項

### ✅ やること
- 文字起こしに書かれている内容のみを記載する
- 不明点・曖昧な点はすべてユーザーに質問する

### ❌ やらないこと
- 自分の解釈や憶測で内容を補足しない
- 一般的な知識で「こうだろう」と付け足さない

### 🔇 スルーする箇所
- `[音楽]` の箇所は常にスルー（説明がない部分）

## 確認フロー
1. 文字起こし受領
2. 要約作成 + タイトル提案 + 不明点の質問リスト
3. ユーザー確認・フィードバック
4. 修正反映
5. 最終確認でOKが出たらファイル保存

## 重要
このコンテキストは今後の開発における会話の基盤となるため、正確性を最優先とする。
