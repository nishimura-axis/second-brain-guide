## 静止画クリエイティブの構造

静止画は以下の3つの要素で構成される：

1. **見出し** - 1番上のテキスト（〇〇を探しています等）
2. **スクリプト** - 見出し以外のテキストすべて
3. **型** - デザインテンプレート

---

## 1. 型（デザインテンプレート）

### 型の大分類：2軸

| 分類 | 特徴 |
|------|------|
| **綺麗目** | 装飾が少ない、シンプル構成、テキストを読ませることを意識、明朝体、文章感がある |
| **ポップ目** | 装飾が多い、イラストなど入れる、デザイン性強い、ゴシック体、カジュアル |

### 綺麗目とポップ目の違い

| 項目 | 綺麗目 | ポップ目 |
|------|--------|----------|
| CTRの取り方 | 見出し・文章で取る | デザインで取る |
| フォント | 明朝体 | ゴシック体 |
| 印象 | 硬い、フォーマル | カジュアル |
| モーション | 少なめ | 真ん中が動くなど多め |
| 効果音 | あまり入れない | 冒頭効果音などこだわる |
| ターゲット層（仮説） | 顕在層 | 潜在層 |

### 型検証の注意点

- 綺麗目とポップ目はキャンペーンを分ける
- ポップの中に綺麗目を入れると綺麗目が勝ったことが1度もない
- 分けた時になぜか綺麗目も別に取れる
- テイスト（訴求軸）が違うのでキャンペーンを分ける
- 最終的に両方強いものができたら、その戦いはさせない（どっちも強いにする）

### ポップ目に効果音やモーションを入れる理由

- 動きがあるものに最初に目が行く
- ユーザーの耳でも目でも聞かせる仕組み
- 反応率が良い

### 綺麗目に効果音やモーションを入れない理由

- デザインの全体の雰囲気を壊さない
- 元がシンプルなので雰囲気とミスマッチするケースがある
- 反応率が良くないケースがある

## 2. 型を作る時に意識する3つの概念

### ① 視線誘導

**Zの法則**
- 人間は文章をZ字の順で読む
- 左上→右上→左下→右下

**Fの法則**
- 文章においてF字の順で読む
- 人間は無意識下で自分に興味ある部分だけ読む
- 重要なオファーは左側に配置する（右端に配置しても読まれにくい）

**視線誘導のポイント**
- 動きがあるものに最初に目が行く
- 真ん中か左上に動くものを置く

**文字色と文字の大きさも視線誘導**
- 人間はブロックの大きさが違った場合、でかいところから読む
- 大事なテキストを読ませたい順に文字サイズを設定する

**読む順番（文字サイズ・色）**
1. 黒の大文字（1番最初）→ 見出し
2. 赤の中文字（2番目）→ オファー
3. 黒の中文字（3番目）→ 強調したい部分
4. その他

**マーケティング的に1番最初に読ませるべきもの**
- 見出しとオファー
- この2つを1番目・2番目に映るようにする
- 1→2という順で読ませると他も興味できて読んでしまう

**太字の使い分け**
- 赤字太字：オファーと限定性
- 黒太字：4-5行目の強調したい部分、料金比較、相場感・お得感

**限定性のテキスト**
- 基本赤字太字
- ただし文字はちっちゃい
- 例：3月末まで、こちらからの申し込み限定、先着何名限定

**1番下のテキスト**
- 赤字にしちゃうとバランス感が崩れる（視張が強くなりすぎてそっちに目がいく）
- 黒の太字でバランスを抑える

### ② 視認性（可読性）

**基本**
- デザインは文章を邪魔しない
- あくまでも読ませたいのは文章
- 可読性をめっちゃ上げる

**余白の重要性**
- 図形と文章が近すぎると読めない
- 空白をしっかり作る
- 圧迫感をなくす

**ブロック間の余白**
- 1-3行目、4-5行目、料金内容詳細、クロージングなどブロックがある
- ブロック間の隙間は全部一緒にする（統一する）
- 一箇所だけ狭いとマジで読みづらい

**テキストが長い場合**
- 下の行を削って見やすくする
- スペースを作って文字の横幅を広げる
- 余白の幅は全部統一

**余白の取り方**
- ネットで調べて世に出てるサイトを見る

### ③ デザイン性

**デザインのコンセプトの持ってき方**
1. イベント
2. 季節
3. 見出しの強いワードから

基本は季節・イベントから。見出しで強い技から取ってこれないケースが多い。

## 3. 型のデザイン：購買意欲とイベントの関係

### 購買意欲が上がる3つの要因

1. イベント性
2. ボーナス
3. 連休

### 型の強さ順（1年間200型以上作った結果）

1. **連休 × イベント** ← 最強
2. **連休**
3. **イベント単体** ← 弱い

### イベント単体が弱い例

- ハロウィン
- バレンタイン
- ひな祭り

これらはクソ雑魚い。マジで作らない。

### なぜ連休が強いのか

- 人間は次の休みを結構意識する
- 次の夏休みどうしようか、次の冬休みどうしようか、次のゴールデンウィークどうしようかの思考
- 脳がもう次の休みのことを考えている

### 連休を打つタイミング

- 連休直前ではなく、連休を脳が意識し始めた瞬間に打つ
- 先行者利益を取れる

**例：夏の型**
- 4月の夏：バツ（1個手前にゴールデンウィークがあるから脳がそっちにある）
- 5月の夏：いい（5月が終わった後の次の楽しみは夏休み、間に何もない）

**例：ゴールデンウィーク**
- 4月はゴールデンウィークっぽいものが強い
- バーベキューポップ、川、こいのぼりなど

### 連休 × イベントで最強のもの

- 夏休み
- クリスマス
- 年末年始
- 新生活

### 1個前のイベントはダメ

- 9月にクリスマス → 売れない（脳がまだ夏の残りで冬を意識してない、9月は暑い）
- 10月にクリスマス → 売れる（寒くなってきた、ハロウィンよりクリスマスの予定を立てる人が多い）

**気温も関係する**
- 9月は暑いから冬という意識にまだ脳がなっていない
- 寒くなってきた10月にクリスマスを打つ方が取れる

### 季節のイベントより強いもの：その時期を想像するもの

- バレンタイン時期 → バレンタインは反応率うんち
- 代わりに「みかん」「鍋」がめちゃめちゃ強かった

**特産物が当たるとずっと伸びる**
- みかん：季節のイメージがある + 他社もやってない独自性
- 鍋：他社がやってない + 冒頭効果音でグツグツグツって音が引きになった

**今の時期（3月）**
- いちご狩りの時期
- いちごが上位

## 4. 見出し

### 見出しのリサーチ元

チャットで共有されたリソースから探す。

### 見出しの特徴

- 見出しは変数として小さい
- 1個当たったらスクリプトと型に力を入れた方がいい
- 見出しで惚れる時代は3年前にあったが、やり続けた結果ガチで意味ないことに気づいた
- 新商材・新ジャンルやる時しか見出し検証はあまりやらない

### 静止画のずらしは取れない

**動画との違い**
- 動画：ずらしで取れる
- 静止画：ずらしで取れない

**理由**
- 大枠が似てたら最初に当たったやつ以降は取れない
- 素材が変わらないから同じ広告だと思われる
- 配信層が一緒になる
- 自社同士で両方ROASが悪くなる
- アカウント変えても一緒

**ずらすなら印象ごと変える**
- 捉え方が違うレベルで変える必要がある
- 例：「近血の男性探してます」→「お金溜まってから脱毛する男性探してます」（印象が違う）
- NG例：「近血の男性探してます」→「お金ない男性探してます」（印象が同じ）

**感情が逆になるようなずらし方**
- 例：「クマがひどい女性探してます」（コンプ訴求）→「クマがなくなっても大丈夫な女性探してます」（逆の言い方）

### 見出しの軸出し

- ユーザーインサイトを広く見る
- この人たちってどういう考え方を持ってるんだっけ？
- メインターゲットってどういうものを欲しがるんだっけ？

**商材理解からの見出し軸出し例**
- ニキビメニューにピコレーザー（色素沈着を消すもの）がある
- シミ取りのピコレーザーが鬼ほど取れている
- ニキビ跡によるシミ、ニキビ跡による色素沈着という言い方で似たもの取れるのでは
- ユーザーインサイトがピコレーザー・シミの方にペインを持てる人が多いからニキビ側に映す

## 5. スクリプト

### スクリプトの構成（4部構成）

| 部分 | 内容 |
|------|------|
| 1-3行目 | 見出し的な導入部分 |
| 4-5行目 | ユーザーインサイトを刺す箇所（1番頭を使う） |
| 料金内容詳細 | 特典・特徴・料金 |
| クロージング | 申し込み誘導（1-2行） |

### 頭を使う順

1. 4-5行目（1番頭を使う）
2. 1-3行目
3. 料金内容詳細
4. クロージング（横展開するケースが多い、あまり頭使わない）

### 太字だけ読んでも分かるようにする

- 人間はおそらくテキスト全部を見ていない
- 太字だけ読んでもなんとなく分かるようにする
- 安いよね、限定だよね、特典こんなのあるよね、がそれだけで伝わるように

### 太字の使い分け

| 色・太さ | 使う箇所 |
|----------|----------|
| 赤字太字 | オファーと限定性 |
| 黒太字 | 4-5行目の強調部分、相場感・お得感、ユーザーのベネフィット |

### 4-5行目のリサーチ元

- 競合の動画
- クラウドワークスリサーチ
- ユーザーリサーチ（ガールズちゃんねる、TikTokのコメント欄、Twitter）
- 記事・LP
- 見出し（売れた見出しを4-5行目に持ってくるケースあり）
- 社内の売れてるクリエイティブ
- 違う商材から持ってくる（例：TCBの「あなたをもっと若々しくさせるサポートさせてください」→ カーブスにも使える）

### 4-5行目の重要性

- 1番ユーザーインサイトを刺しに行ってるテキスト
- ここでユーザーがどういう反応をするか調べる
- 変数として1番大きい
- 最初のスクリプト検証は4-5行目だけずっと試す

### 競合動画から持ってくるだけでは取れなくなる

- 最初は取れる（ナハトが検証した訴求軸の大量のものをそのまま持ってくればいい）
- どこかのタイミングで競合のPDCAよりうちのPDCAが早くなった時に動画から拾えなくなる
- その状況でも拾い続けるとマジで取れない
- ユーザーインサイトじゃなくてテクニカルで取ろうとしちゃってるから意味ない
- 4-5行目はちゃんとファンダメンタル（インサイト）で刺しに行かないとクリエイティブは取れない

## 6. 検証方法

### 検証の鉄則

**変数は必ず1つにする**
- その他は1番最強か安牌でキャンペーン・クリエイティブを構成

### 変数の種類

1. 見出し
2. スクリプト（4-5行目）
3. 型
4. 記事
5. LP
6. ピクセル

### 検証の例：見出し検証

- 見出しだけ変える
- 型：白黒サイ（固定）
- スクリプト：「無理な追加料金さございません」（固定）

### 検証の例：スクリプト検証

- 4-5行目だけ変える
- 他は全部固定

### 検証の例：型検証

- 型だけ変える
- 見出し：競合で1番取れてるもの（固定）
- スクリプト：4-5行目「無理な追加料金さございません」（固定）

### 複数軸の検証を同時に走らせる場合

**例：特典 vs 特徴検証 + 4-5行目検証**

1. **キャンペーンA**：4-5行目検証（料金内容は特典で固定）
2. **キャンペーンB**：特典 vs 特徴検証（4-5行目は固定）

**結果の組み合わせ**
- キャンペーンAで4-5行目の3番が強かった
- キャンペーンBで特徴の方が強かった
- → 4-5行目の3番 + 特徴 を組み合わせれば最強

### 検証期間

- 3日以上
- 数値計測以上

### 検証のフェーズ

| フェーズ | 内容 |
|----------|------|
| 第1フェーズ | 最強を見つける検証をする |
| 第2フェーズ | 検証の結果から展開する |
| 以降 | 検証 + 展開を同時に1週間内で行う |

- 初週だけ検証
- 2週目からは検証と展開を同時に行う
- 課題となりそうな箇所だけ検証する

## 用語解説

| 用語 | 説明 |
|------|------|
| 型 | クリエイティブのデザインテンプレート |
| スクリプト | 見出し以外のテキストすべて |
| 見出し | 静止画の1番上のテキスト |
| 型リスト | 静止画チームで過去に制作した型の一覧 |
| 綺麗目 / ポップ目 | 型の大分類（2軸） |
| 4-5行目 | スクリプトの4行目と5行目（ユーザーインサイトを刺す箇所） |
| 1-3行目 | スクリプトの1〜3行目 |
| スクエア | スクエアサイズ（正方形）のクリエイティブ |
| 明朝体 | フォントの種類（硬い、フォーマル） |
| ゴシック体 | フォントの種類（カジュアル） |
| 特典 | オファー内容 |
| 特徴 | 商材の特徴・メリット（USPに近い） |
| クロージング | スクリプトの最後の部分（申し込み誘導、1-2行） |
| 横展開 | 売れたクリエイティブを元に展開すること |
| Zの法則 | デザインにおける視線誘導の法則（Z字に読む） |
| Fの法則 | 文章における視線誘導の法則（F字に読む） |
| デイリーサーチ | 社内の日次リサーチ共有（SNSの広告やオーガニック投稿からアイデアを参考にする） |
