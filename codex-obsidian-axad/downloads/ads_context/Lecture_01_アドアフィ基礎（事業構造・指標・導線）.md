## 1. アクシスの沿革

- 安達さんがナハトの代表で、安達さんと正哉（AXIS代表）が大学の先輩後輩の関係
- 彼らは一緒にビジネスを始めた
- 最初から法人は分けている（別会社だが同じことをやっていた）
- 元々インフルエンサーマーケティング（インフルエンサーアフィ）から始まった
- インフルエンサーアフィを2〜3年やっていた
- その途中からアドアフィに参入（3年前ぐらい）
- 過去に脱毛サロンをやっていた（勘太が店長、畳んだ）
- 現在、翔平が新規事業（ファンマーケ）をやっている

## 2. ナハトとの関係性

- ナハトとは協業のような近しい関係
- 昔はアドアフィもナハトと一緒にやっていた
- クリエイティブチェックをナハトとして出す場合がある（クライアント側がアクシスを知らない場合）
- **注意**：ナハトとしてやってるのにアクシスの人間としてチェック出すとクライアントが混乱する

### ナハト商流とアクシス商流

| 商流 | 説明 |
|------|------|
| ナハト商流 | クライアント→ナハト→アクシスに下ろしている案件（ナハトを経由して実施） |
| アクシス商流 | アクシスが直接クライアントから引いている案件 |

※実際にはクライアントとAXIS/ナハトの間にASPがいる

## 3. 予算運用とアドアフィリエイトの違い

### 予算運用（一般的な広告代理店）

- クライアント（大手メーカー等）からコンペで予算を獲得
- 月1億〜2億などの大きい予算をもらう
- その予算内でクリエイティブ・バナー・広告費を出す
- 自社持ちの費用は基本的にない
- 運用額の20%を手数料としてもらう（例：1000万消化→200万が手数料）
- 結果が出なくても予算を消化すれば手数料は入る

### アドアフィリエイト

- 成果報酬（1件いくら）
- 広告費は自社持ち
- 単価の中でCPAを抑えて獲得し、差分が利益
- 実績がなくても獲得力があれば伸ばせる
- ベンチャーでも早い段階で月1億〜2億の売上を作れる

---

## 4. アフィリエイトの基本構造

```
クライアント → ASP → 代理店（一次代理店）
```

- ASPは様々なクライアントと繋がり、案件を持っている
- ASPは案件を代理店に下ろす
- 例：クライアントが1件1万円でASPに卸す場合、基本的にASPは10%程度抜いて9000円で代理店に下ろす
- ASPは手数料で利益を得るが薄利（10%）
- ASPも基本的に自社運用をやっている
- 最近の傾向：自社運用が強いASPしか生き残っていない

### 2次代理店からさらに下ろす場合

- 2次代理店からさらに別のメディアに下ろす場合もある（3次、4次）
- 間に挟まれるほど中間手数料を取られる
- できるだけ上流に近い方がいいが、近すぎると面倒な時もある

---

## 5. 直クラ（ASPを挟まない直接取引）

### 例

- DIO：直クラでやっている
- TCB：ASP（エンターキー）を挟んでいる

### 直クラのメリット

- 手数料が抜けないので高単価で下ろせる
- 要望を伝えやすい

### 直クラのデメリット

- コミュニケーションコストが重い
- タグ周りやアフィの知識がないクライアントだと要望が通しにくい

### ASPを挟むメリット

- 条件調整が済んだ状態で案件を下ろしてくれる
- コミュニケーションコストが最小限

### 判断基準

- 中長期で付き合いたいクライアントなら直クラが良い場合もある
- 案件や条件による

|------|----------------------|
| クライアント | 代理店が獲得したCVでユーザーが購入（EC）または来店・契約 |
| 代理店 | 単価に対してCPAを安く取れた差分が利益 |

## 7. アドアフィが存在するジャンル

### リード

- **有形（店舗型）**：うちがメインでやっているところ
- **無形**：債務、不動産など店舗を持たないタイプ

### EC

- サプリなど
- 特徴：単価が低い、他社が過激な広告を出している、薬機法の規制が厳しい
- 若干シュリンク気味の市場

### うちの状況

- 全ジャンル触っているが、今はほぼ有形リードに寄っている
- 1年前まではECもやっていた
- メンズクリアやストラッシュみたいな案件がベースで会社が大きくなった

## 8. 媒体

### うちで実施している媒体（5媒体）

- Meta
- Google
- TikTok
- LINE
- X

### 媒体の優先順位と展開

- 運用の流れ：基本的にFB/TikTokから始めて、良かったものをYouTube/Xに展開
- 当たったクリエイティブは基本的に他媒体でも取れる（媒体特性はあるが素材がはまっていれば利益出る）

### その他の媒体（参考）

- Yahoo
- スマートニュース
- Microsoft広告
- YahooやスマートニュースはECのボリュームが多い
- SNS系で有形リードが盛り上がっている

---

## 9. 利益拡大の2軸

1. **案件の展開**：同じジャンルの別案件に同じ素材で展開
2. **媒体の展開**：当たったクリエイティブを別媒体に展開

どちらもできたら利益を最大化できる

## 10. アカウント構造

### TikTok、LINE、Google

- 親アカウント（TikTokだとビジネスセンター）の中に各案件ごとの広告アカウントが紐づいている
- 親アカウントの共通ID/パスワードで全アカウントにアクセス可能

### Meta（特殊）

- ビジネスマネージャー（親アカウント）の中に広告アカウント（案件ごと）が紐づいている
- 共通ID/パスワードは存在しない
- SNSとしての個人アカウントにアクセス権を与える形式
- Meta社のポリシー：普段から自社のSNSアカウントを使っている人にしか広告アカウントを触らせない
- SNSアカウントは実名登録・2段階認証のセキュリティ設定が必要
- 設定していないと不審なアカウントとしてBANされる

### BANについて

- 広告アカウント単体がBANされることが多い
- ビジネスマネージャー自体がBANされることもある
- 一番きついのはSNS個人アカウントが完全凍結されること
  - 作り替えが必要だが、1つのIPアドレスで複数アカウントにログインすると不審なアカウントになる
  - IPアドレスや端末を変える必要がある
- **注意：1つのPCで別の人のFacebookにはログインしないこと**

## 13. 広告本部の動き方（4段階）

### 1. 案件選定

- 担当：上長陣（勘太、木口など）
- ASPやクライアント直でフロントとして動く
- 条件決め、交渉、案件を引っ張ってくる

### キャップについて

- キャップ：月に何件取っていいかの上限
- 例：月100件までで単価2万なら売上上限200万
- TCBは月に数千件もらっているので大きい売上になる

### 2. 準備段階（案件決定後）

| 作業 | 担当 |
|------|------|
| クリエイティブ作成 | 運用者 |
| 記事作成 | ライティング部（記事部）※貸し記事もある |
| 繋ぎ込み（アカウント開設・リンク発行・タグ設置） | アド管理部 |

### 3. 出稿

- 出稿チーム（アド管理部内のアルバイトチーム）が手伝う
- 運用者が精査（毎日朝7時頃から）

### 4. 分析・改善

- 運用データを踏まえてデータ分析
- 良かったものは展開、悪かったものは改善
- 分析→制作のPDCAを回す

## 14. 運用者が最もインパクトを出せる業務

- **クリエイティブ制作**と**分析**
- 特に新しい訴求（まだ市場に出ていない見せ方や構成）を生み出した時に先行者利益を取れる
- 部署を分けて運用者がクリエイティブ制作と分析に時間を使えるようにしている
- 元々は1人の運用者が全部やっていた（西村や勘太など）

---

## 15. CVまでの導線

```
クリエイティブ → 記事 → LP → サンクスページ（CV発生）
```

### LPを挟む理由

- 教育しないとユーザーが誤解したまま来店し、契約に繋がらない
- 例：「6ヶ月0円」だけ見た人が無料だと思って来店→実際は頭金何十万必要→契約しない
- 記事でちゃんと教育できていればLPを飛ばしてフォームからサンクスページへの導線もある
- LPがめっちゃ作り込まれている場合はアンケートだけでもいい場合もある
- クリエイティブからいきなりLPに行くパターンもあり、これを**直LP**と呼んでる（EC特に多い）
- うちは基本記事を挟む
- **LPは基本的にクライアントが作るもの（あまりいじれない領域）**

---

## 16. アンケート記事について

- アンケートは集計目的ではなく、ユーザーに頭を使わせて教育させるため
- クリックさせる行動を起こさせることで、主体的に読んでくれる
- 自分で手を動かして進んでいくと、自然な流れで予約まで至る
- アンケート記事は3年前ぐらいに見つかった手法

---

## 17. イラスト記事について

- 漫画で読めるから頭に入りやすい
- 長くてもユーザーが遷移してくれる
- 漫画がフックになっている

## 18. 指標

| 指標 | 説明 |
|------|------|
| ROAS | 売上 ÷ 消化金額。高いほど利益が大きい |
| CVR | LP→サンクスの遷移率 |
| MCVR | 記事→LPの遷移率（Micro CVR） |
| CTR | クリエイティブ→記事のクリック率 |
| CPA | 獲得単価。1CV発生にいくらコストを使ったか |
| CPM | 1000回表示あたりのコスト |
| CPC | クリック単価。CTRと反比例の関係 |
| Pixel | Meta Pixelなど、CV計測用のタグ |

- CVR、MCVRは案件と媒体ごとに適正値がある（過去の運用データから）
- CPCも媒体と案件で適正値がある（例：TCBのメンズクリアはCPC200円が目安）
- CPCはクリエイティブだけでなく運用の要素も入る

---

## 19. 導線の計算例

**条件**：目標CPA1万円、単価2万円、MCVR10%、CVR10%、CTR1%、CPM1000円

| ステップ | 必要数 |
|----------|--------|
| CV1件発生 | LP→サンクス10%なのでLPに10人必要 |
| LPに10人 | 記事→LP10%なので記事に100人必要 |
| 記事に100人 | CTR1%ならクリエイティブを1万人に表示 |
| 1万人表示のコスト | CPM1000円なら1万円 |

→ CPA1万円で達成、ROAS200%

## 20. 数値が悪い時の改善の考え方

### CPCが高い場合（例：500円）

- クリエイティブが良ければCTRが上がりCPCは下がるはず
- クリエイティブの改善が必要

### MCVRが悪い場合

- クリエイティブと記事の訴求に差分がある
  - 例：クリエイティブでエロ訴求→記事は硬い内容→ギャップで離脱
- 運用のターゲティングミス
  - 例：若者向けの記事・クリエイティブなのに配信は年配層

### 要因の種類

- **内的要因**：クリエイティブ要因、運用要因
- **外的要因**：同じターゲットに競合がより良いクリエイティブを出している

### 差分分析の視点

- クリエイティブ視点
- 記事視点
- LP視点

---

## 21. CPMが高騰する要因と対策

### 外的要因

- 12月、3月など企業の決算期に大きい企業が広告費を大量に使う
- 同じターゲットに大きい予算を出すところがいると、入札単価が上がる
- 広告配信はオークション形式

### 内的要因

- クリエイティブがはまっていない
- キャンペーンの設定ミス（例：男性向けなのに女性に配信）

### 対策

- ターゲットをずらす
- ピクセルを変える
- キャンペーンを作り替えて面を変える

---

## 22. 運用者として大事なポイント

- 導線全体を意識する（クリエイティブだけに目を向けない）
- 記事・LPの内容を読み込む
- クリエイティブと記事の整合性を持たせる
- どこがボトルネックか見極める
- 指標（CPC、MCVR、CVRなど）をパッと頭に浮かぶレベルで覚える
- 各指標の相関性を理解する
- ROASが高い/低い時にどの数値が影響しているか解像度高く持つ

---

## 23. クリエイティブの重要性

- クリエイティブが一番インパクトがある（特に動画媒体）
- クリエイティブが良くなければ記事・LPがいくら良くても到達しない
- 入り口として非常に大事

---

## 24. 予算について

- 1日でいくら使うかの上限金額
- 目標単価に比例する
- 設定したクリエイティブをいくら流すかの設定
- 24時間で使い切るように媒体側が配信

---

## 25. CPMの決まり方

- 媒体側・AIが決めている
- 明確に「いくら」と決まっているわけではない
- 市況による
- 同じターゲットに出したいメディアが多いほど入札単価が高くなる
- 予算を張ってくれるところ、成果が合っているところに優先配信
- 数値が悪い広告は良くない広告として認識され、いい面に配信されにくい

---

## 26. その先の話（参考）

- ユーザーはサンクスページ到達後、来店→契約でクライアントの売上になる
- 無料オファーでないのに無料のように訴求すると、契約に繋がらない
- クライアントの売上が発生しないと案件が潰れる
- 訴求の調整はフロントの人がやっている
- **運用者としてはサンクスページ到達までの導線を意識すればOK**
