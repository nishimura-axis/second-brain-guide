## 1. 広告本部の業務フロー

```
案件選定 → 制作 → 運用（出稿） → 分析 → 制作に戻る
```

1. **案件選定** - どの案件をやるか決める
   - 基本的には店舗系の案件（クリニックなど）
   - TCBがメイン（クマ再生注射、ヒアルロン酸注射などいろんなメニュー）

2. **制作** - クリエイティブと記事を作る
   - クリエイティブ：動画やバナー（実際に媒体に配信するもの）
   - 記事：その先のページ

3. **運用** - 制作ができたら運用に入る
   - 運用に入る時にやるのが「出稿」という業務

4. **分析** - 運用した次の日から結果が出てくる
   - 結果を見て分析
   - いいものはさらに展開、悪いものは改善
   - また制作に戻る

### 運用者の業務

- 制作→分析をひたすらぐるぐる回す
- 1運用者は大体5〜6案件を扱っている
- 多い人だと1日に100個とかのクリエイティブを作っている
- 100個分のクリエイティブを出稿すると結構時間かかる（1日2〜3時間）

### 会社にとって1番大きな売上を生む場所

- **クリエイティブ**
- いいクリエイティブができた時に1番売上につながる
- 成果報酬なので、1件の獲得コストをどれだけ抑えれるかが重要
- 後ろに続くLPとかの内容がどんなに良くても、最初に目に触れるクリエイティブがユーザーの興味を引けていなかったら後ろにたどり着かない

### 出稿が生まれる背景

- 運用者をより制作に注力してもらうために、出稿を依頼するフローができている
- 出稿を巻き取ってもらえることで、より制作に注力できるようになって売上が出ていく
- **すごい重要な役割**

## 2. 売上発生までの流れ

```
ユーザー（インスタを見ている人）
→ クリエイティブ（広告）が流れてくる
→ クリックすると記事に飛ぶ（自社で作っている）
→ 記事を見て良さそうだなと思ったら次のボタンをクリック
→ LP（クライアントが作っている予約フォームのページ）
→ 予約完了
→ サンクスページ（予約ありがとうございましたみたいなページ）
→ ここに到達して初めてコンバージョン（CV）が発生
```

- コンバージョン = 予約完了したよというシグナル
- 単価2万円だったら1CV = 2万円の売上

### 利益の作り方

- 1件獲得するまでにいくら広告予算を使ったか（消化金額）
- 例：消化金額1万円で1CV発生 → 単価2万円 - 消化1万円 = 利益1万円
- 消化金額5000円で1CV発生 → 利益1.5万円

### 配信設定の重要性

- 配信設定が適切に行われないとCVが発生しない
- 例：男性用の脱毛サロンの広告を女性に配信してしまったら、クリエイティブが無駄になる
- 依頼しているから全部任せるわけではなく、双方の確認は必要
- **最初のうちは速さより正確さ**（依頼内容通りにキャンペーン設定ができているか）

### 実績（参考）

- TCBだけでも月に1万件ぐらい取れている
- 広告から予約する人はいっぱいいる

## 3. 出稿とは

- Meta、TikTok、Xなど主にこの3媒体の配信設計を行うこと
- どんな広告を流したいか
- どんな人に配信したいか（何歳、どの地域に住んでいるか、性別など）

### 覚えて欲しいこと（3つ）

1. **キャンペーン**
2. **広告セット**
3. **広告**

### キャンペーン・広告セット・広告の構造

```
キャンペーン
└── 広告セット
    └── 広告
```

| 階層 | 設定内容 |
|------|----------|
| キャンペーン | 箱。名前をつけたり、予算の設定（1日でいくら予算使いたいか）をする場所 |
| 広告セット | オーディエンスの設定（年齢、性別、地域）、どのユーザーに届けたいかを設定する場所 |
| 広告 | 実際に配信したいクリエイティブと遷移先の記事を設定する箇所 |

- これらが全部キャンペーンの中に含まれて設定されたら初めて媒体に広告が流れるようになる
- 基本的には翌日の0時から流し始める

---

## 4. 設計のパターン

### パターン1：1キャンペーン・1広告セット・複数広告

- 例：3つのクリエイティブ（A、B、C）を作った
- どの訴求が1番良いか検証したい
- 1つの広告セットに3つ入れてどれが1番効果的か検証する

### パターン2：1キャンペーン・複数広告セット・各1広告

- 例：Aの広告が1番良かった
- Aの広告をさらに年齢別に分けて配信したい（18歳のあなたへ、19歳のあなたへ）
- 1個1個の広告セットごとにクリエイティブを1個ずつ分ける

### 基本的な2パターン

1. 同じ設定の広告セットで複数クリエイティブを検証するパターン
2. 広告セットごとに配信年齢を分けて、それぞれに合ったクリエイティブを配信するパターン

### 特別な設計パターン

- 特別な設計のパターンもある
- 追い追い覚える

## 5. 出稿業務の大まかな流れ

1. **運用者から依頼が入る** → スタート
2. **担当の振り分け** - アルバイトの振り分け
3. **シートを確認** - 何の案件か、どの媒体かの情報を確認
4. **クリエイティブをダウンロード** - Googleドライブで納品される
5. **媒体にアップロード** - ドライブのリンクそのままでは配信できないので、媒体側にアップロードする必要がある
6. **記事の複製・リンク設定** - ベースの記事があればそれを複製して配信用にリンク設定
7. **媒体管理画面上で設定**

### 出勤時間について

- 11時出勤にしているのは、運用者が出勤してすぐクリエイティブができるわけではなく、出勤してからクリエイティブが上がってくるから1時間ずらしている

### 基本は待ちの姿勢でOK

- 明らかに何も入っていなくて手が空いている時は声かけてほしい

## 6. 出稿業務において大事なこと

### なんでキャンペーンごとにアフィリンクを分けるのか

- アフィリンク = LPのところ
- キャンペーン1個1個作るごとに遷移先の記事とか全部分けている
- 中身は同じだけどURLが違う

**理由：効果測定を正確にするため**

- 2種類のキャンペーンを運用していて、最終的な遷移先のページが同じだった場合、CVが発生してもどっちのキャンペーンについたか測定ができなくなる
- どのクリエイティブでCVがついたかはすごい重要な情報
- それを識別するためにキャンペーンを分けている

### アフィリンクとは何か

- アフィリンクとLPのURLは違うが、飛び先は一緒
- 記事の中に入れて欲しいのはアフィリンクの方
- アフィリンクだと管理画面でどの広告で件数が発生したかの成果情報が全部見れる
- ただのLPだと計測ができない（効果があっても成果が発生しない）
- **計測するためにアフィリンクを設定している**

---

## 7. 意識して欲しいこと

### 分からないことがあったら何でも聞く

- アド管理メンバー
- 他のアルバイト
- 依頼者（運用者）
- アルバイト同士もコミュニケーション取っている

### 不明点をそのままにしない

- なんとなくこうだろうでキャンペーン作っちゃって、実際に配信したいものとずれた場合に損失が生まれる
- それによって設定した人を責めることは絶対ない
- あくまでその運用者の責任
- **依頼受けた側も不明点はそのままにせず必ず明らかにする**

### 最初のうちは速さは全然意識しなくていい、正確にやること

- 慣れてきたら勝手に早くなってくる
- 焦らずで大丈夫
