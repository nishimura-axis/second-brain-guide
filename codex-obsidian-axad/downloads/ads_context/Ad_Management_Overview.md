# 🛠️ アド管理 業務内容

> ⚠️ **注意**: 業務内容と定期タスクは2026年1月時点のものです。閲覧時点で変更されている可能性があります。

---

## ミッション

> **広告本部の火力最大化 ＝ リソース捻出による制作の質・量の最大化**

プレイヤーがCR制作（コア業務）に注力できる環境を作ること。

### なぜCR制作が最重要か

アド事業部はAXIS全体のキャッシュエンジン。その中で最もレバレッジがかかるのが**クリエイティブ制作**。

- 市場にない訴求や見せ方ができた時、先行者利益で莫大な利益が出る
- クリエイティブが良くなければ、記事・LPがいくら良くても到達しない
- 入り口として非常に大事（Lecture 01参照）

### アド管理の役割

運用者が制作と分析に注力できるよう、以下を担う：

- **ノンコア業務の巻き取り**
- **業務の自動化**
- **依頼フローの構築**（アルバイトでも対応できる仕組み化）

### 組織上の位置づけ

バックオフィス寄りだが、コーポレート系（総務・労務・経理・人事）とは異なる。

```
バックオフィス ←【アド管理】→ フロント（運用）
```

**事業部専任のサポートチーム** = 運用者の業務フローの**各つなぎ**を担当

```
案件選定 → 制作 → [つなぎこみ] → 出稿 → 運用 → 分析 → [数字合わせ]
                    ↑                              ↑
                アド管理                        アド管理
```

## 緊急度×重要度マトリクス

アド管理部における業務の優先順位付けの考え方。

参考: [優先度マトリクス](https://www.jooto.com/contents/priority-matrix/)

### マトリクス概念図

```
        重要度
          ↑
    ┌─────────┬─────────┐
    │  黄色   │   赤    │
    │ 生産性  │ 損失回避│
    │ 向上系  │ ＆緊急  │
    ├─────────┼─────────┤
    │  青     │ オレンジ │
    │ 低優先  │ 損失回避│
    │         │ ＆非緊急│
    └─────────┴─────────┘ → 緊急度
```

### 4象限の定義

| 色 | 分類 | 定義 | 具体例 |
|----|------|------|--------|
| 🔴 赤 | やらないと損失が生まれる＆緊急度が高い | 即対応必須。放置すると制作の機会損失に直結 | 精査/分析に関わるシートエラー、デカモニの滞り、媒体のエラー |
| 🟠 オレンジ | やらないと損失が生まれる＆緊急度が低め | 期限内に処理。放置すると運用の機会損失に直結 | つなぎこみ（リンク発行、アカウント開設など）、定常業務 |
| 🟡 黄色 | やらなくても回るけど生産性が高まる | 常に推進。広告本部の攻撃力最大化に直結 | 出稿フローの整備、自動化関連、属人化の解消（マニュアル作成） |
| 🔵 青 | やらなくても困りはしない | 余裕がある時に対応 | 既存スプシの改修（膨らみなき）、ページ一覧の整理 |

### ミッション達成のための優先順位

```
赤・オレンジを円滑に回す（損失回避）
        ↓
黄色を常に推進し続ける（生産性向上）
        ↓
【結果】広告本部の攻撃力最大化を実現
```

## 何をタスクとするか？

### 『問題（課題）』と『思わしくないこと』の違い

```
現実 ──────→ 理想
      ↑
   広告本部の方針
  （アド管理のミッション）
```

| 種別 | 定義 | タスク化 | マトリクス上の位置 |
|------|------|----------|-------------------|
| **問題（課題）** | 理想に向かう方向性（広告本部の方針）に沿って生まれる「思い通りにいかないこと」 | **タスクにする** | 赤・オレンジ・黄色 |
| **思わしくないこと** | 理想に向かう方向性に**沿わない**ところで発生する「思い通りにいかないこと」 | **タスクにしない** | 青（またはタスク外） |

### 判断基準

> **広告本部の方針に沿わないところで発生する問題は、『問題』とする必要がない ＝ タスクではない**

アド管理のミッションは「広告本部の攻撃力最大化」なので、そこに貢献しないものはタスク化の優先度を下げる（または対応しない）。

### 具体例

| 事象 | 判断 | 理由 |
|------|------|------|
| 精査シートのエラーで運用者が分析できない | 🔴 問題（赤） | 分析が止まる→制作の機会損失 |
| リンク発行が遅れて出稿が止まる | 🟠 問題（オレンジ） | 運用の機会損失 |
| 出稿フローが属人化している | 🟡 問題（黄色） | 解消すれば生産性向上 |
| 使わなくなったスプシが整理されていない | 🔵 思わしくないこと | 整理しなくても運用に影響なし |

## 業務カテゴリ（6分類）

### 1. つなぎこみ 🟠

**広告配信開始までの技術的なセットアップ全般**

案件が決定してから運用開始できる状態にするまでの準備業務。

| 業務 | 内容 | 関連講義 |
|------|------|----------|
| リンク発行 | 各代理店/ASP商流のアフィリエイトリンク発行・複製・格納 | Lecture 04 |
| アカウント開設 | 各媒体（Meta/TikTok/LINE/YouTube/X）の広告アカウント開設・ストック管理 | Lecture 02 |
| タグ設置 | LPやサンクスページへのCV計測用タグの設置依頼 | Lecture 03 |
| API連携設定 | ASP管理画面とピクセルの連携設定 | Lecture 03 |
| 発火確認 | CVイベントが正しく発火しているかの確認・テスト予約 | Lecture 03 |

**マトリクス上の位置**: 主に🟠オレンジ（やらないと運用の機会損失）

### 2. 数字合わせ 🔴

**月次の売上確定業務**

月の消化金額と売上を確定させるための突合せ業務。

| 業務 | 内容 |
|------|------|
| データ突合 | 媒体管理画面とASP管理画面のCV数突合、差異確認 |
| CSV修正 | ずれがあった場合のデータ修正 |
| 売上確定 | 経営会議までに数字を確定 |

**マトリクス上の位置**: 🔴赤（期限があり、やらないと経営判断に影響）

### 3. 出稿部管理 🟠🟡

**出稿アルバイトチームのマネジメント**

運用者がCR制作に注力できるよう、出稿業務を巻き取るチームの管理。

| 業務 | 内容 | マトリクス |
|------|------|-----------|
| シフト・勤怠管理 | 出稿部アルバイトのシフト作成、勤怠管理 | 🟠 |
| 教育 | 新人アルバイトの教育、出稿方法の指導 | 🟡 |
| 依頼振り分け | 運用者からの出稿依頼をアルバイトに振り分け | 🟠 |
| キャパシティ管理 | 依頼過多時の全体アナウンス、手すき時の連携 | 🟠 |
| 品質管理 | 依頼内容の不備対応、エラー対応、課題の吸い上げ | 🟠 |
| フロー整備 | 出稿フローの改善、属人化の解消 | 🟡 |

### 4. スプレッドシート管理 🔴🟠🔵

**運用に必要なシートの保守運用**

| 業務 | 内容 | マトリクス |
|------|------|-----------|
| 新規作成 | 月別AXIS総合売上シート、チーム別運用シートの作成 | 🟠 |
| エラー対応 | 精査用シート、月別運用シートのエラー対応 | 🔴（分析に直結） |
| 保守 | 各媒体の広告アカウント整理、過去CSV修正 | 🟠 |
| cobit管理 | Cloud型RPA（Biztex社）のエラー対応、新規ASP追加 | 🟠 |
| 既存シート改修 | 膨らみのない既存スプシの改修 | 🔵 |

### 5. 自動化 🟡

**業務効率化のための仕組み構築**

| 業務 | 内容 |
|------|------|
| スプレッドシート自動化 | GASによるシート処理の自動化 |
| GCP活用 | Google Cloud Platformを使った自動化基盤 |
| ツール開発 | 業務効率化のためのツール開発 |
| RPA | cobitを活用した定型業務の自動化 |
| マニュアル作成 | 属人化解消のためのドキュメント整備 |

**マトリクス上の位置**: 🟡黄色（やらなくても回るが、やれば生産性向上）

### 6. その他 🟠🔵

**上記に含まれない横断的な業務**

| 業務 | 内容 | マトリクス |
|------|------|-----------|
| 全体アナウンス | 出稿部シフトの全体周知、キャパ状況の共有 | 🟠 |
| 請求書支払い | TikTok、X等の請求書処理 | 🟠 |
| Meta請求書突合 | エンターキーから借用したクレカの明細とMeta広告消化金額の突合確認（[[Meta_Invoice_Reconciliation\|詳細]]） | 🟠 |
| 外部連携 | 代理店/ASPとの連絡、タグ設置依頼の調整 | 🟠 |
| BAN対応 | 審査リクエスト、適格性診断対応 | 🟠 |
| ページ一覧整理 | 使わなくなったページ等の整理 | 🔵 |
